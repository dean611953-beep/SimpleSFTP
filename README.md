# SimpleFTP - Windows 轻量 FTP 服务端

让任何人的 Windows 电脑变成一个 **FTP 接收服务器**。

## 场景

**你是接收方**：客户/合作方通过 FTP 工具（WinSCP、FileZilla、FlashFXP 等）把文件上传到你的指定目录。

## 功能

- **一键启动/停止** FTP 服务
- **记录所有上传文件**：文件名、大小、时间、来源 IP、用户名
- **实时日志**：谁在什么时候上传了什么
- **配置保存**：端口、用户名、密码、保存目录
- **支持断点续传**：FTP REST 指令
- **支持被动模式**（PASV）
- **支持目录操作**：CWD、LIST、NLST

## 使用方法

### 第一步：配置

1. **端口**：默认 21，可改
2. **用户名/密码**：客户登录用的凭证
3. **保存目录**：上传文件存放位置

### 第二步：启动

点击 **▶ 启动** 按钮

### 第三步：告诉客户

把以下信息发给客户：

```
FTP 服务器地址: 你的局域网IP
端口: 21
用户名: admin (或你设置的)
密码: xxxxx
```

### 第四步：客户上传

客户在他们的电脑上用任意 FTP 客户端连接即可上传。

## 网络配置

### 局域网使用

直接启动即可，客户在同一局域网内通过你的 IP 连接。

### 外网使用

需要配置路由器端口转发：
1. 进入路由器管理页面
2. 设置端口转发：外部端口 21 → 你的电脑内网 IP:21
3. 客户通过你的公网 IP 连接

## 安全性提示

- **仅在内网使用**时开放
- 外网使用建议：
  - 设置强密码
  - 使用非标准端口（非21）
  - 考虑使用防火墙限制来源IP
  - 考虑升级到 FTPS（SSL加密）

## 系统要求

- **操作系统**: Windows 10/11 (64位)
- **.NET Runtime**: .NET 8.0 Runtime

## 编译

```bash
dotnet restore
dotnet build -c Release
dotnet publish -c Release -r win-x64 --self-contained false -p:PublishSingleFile=true -o ./publish
```

发布的 exe 位于 `publish/SimpleFTP.exe`
